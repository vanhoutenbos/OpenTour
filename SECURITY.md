# Security Policy

## Supported Versions

OpenTour is in pre-release (v0.x). Alleen de nieuwste versie wordt ondersteund met
security updates.

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

Wij nemen beveiligingsproblemen serieus. Rapporteer een kwetsbaarheid via:

1. **GitHub Security Advisory** — ga naar
   https://github.com/vanhoutenbos/OpenTour/security/advisories/new
   (prive melding, zichtbaar alleen voor de beheerders)

### Wat te verwachten

- We werken op best-effort basis, wat inhoud dat je zsm antwoord krijgt.
- We houden je op de hoogte van de voortgang tot de fix is uitgerold.
- Als de kwetsbaarheid wordt geaccepteerd, werkt het team aan een patch en plannen
  we een gecoördineerde publicatie.
- Als het wordt afgewezen, ontvang je een onderbouwing.

### OpenTour is AGPL-3.0

Iedereen mag de code forksen en zelf hosten. Wij stellen het op prijs als je bij
een gevonden kwetsbaarheid eerst contact opneemt voordat je een public fork maakt,
zodat andere gebruikers ook kunnen updaten.
